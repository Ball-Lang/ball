# Upstream protobuf conformance — expected failures for ball_protobuf.
#
# Format: one fully-qualified conformance test name per line (text after the
# first '#' on a line is an inline comment / the runner's failure reason and is
# ignored by the matcher). The conformance_test_runner treats every name here
# as an EXPECTED failure and fails the suite if one unexpectedly PASSES — so
# prune a line the moment its test starts passing.
#
# Scope: the harness registers all three conformance message families —
# protobuf_test_messages.{proto2,proto3,editions} TestAllTypes* — from a protoc
# FileDescriptorSet and drives ball_protobuf's feature-aware binary/JSON codecs.
# Status: 2513 pass, 194 expected-fail (listed below), text-format + unregistered
# message types are reported skipped.
#
# The expected failures are tracked follow-ups, grouped by the feature they need:
#   * Well-Known Types JSON mapping not wired into the descriptor codec:
#     google.protobuf.Any, Struct/Value/ListValue/NullValue, Timestamp, Duration,
#     the *Value wrappers, and FieldMask (the bulk of JsonInput.* failures).
#   * Oneof tracking — descriptors carry no oneof membership, so set-sibling
#     clearing / last-set-wins across a oneof is not modelled.
#   * Singular/repeated MESSAGE merge — repeated occurrences of a singular
#     message field must merge recursively (concat-parse).
#   * Unknown-field retention — unknown fields are dropped, not re-emitted in
#     order (UnknownOrdering / UnknownVarint / MessageSet).
#   * json_name / snake_case field-name handling (e.g. _field_name3) and enum
#     value aliases (allow_alias) — need richer descriptor metadata.
#   * Remaining parse/serialize strictness (non-minimal varints, some UTF-8 and
#     float/double overflow rejections) and unquoted >2^63 integer literals
#     (precision is lost in Dart's jsonDecode before our codec sees the token).
#
# Entries are bare test names (no inline failure message) so they match any
# failure of that test — several reference diffs are order-non-deterministic
# (e.g. Struct map fields). Regenerate after codec changes by running the runner
# without --failure_list and replacing the block below with the failing test
# names, reasons stripped (see conformance/README.md).
#
