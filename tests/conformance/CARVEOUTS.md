# Conformance Carve-Outs

Every `.ball.json` in this directory **must** be regenerable from a Dart source in
`src/<name>.dart` via `dart run bin/generate_conformance.dart` (from `dart/encoder`),
**except** the fixtures listed below. These are the only fixtures allowed to exist
without a matching `src/<name>.dart`. The gate
`dart run bin/check_conformance_sources.dart` (CI: the `ball-freshness` job) fails if
any *other* `.ball.json` lacks a source, or if a name listed here no longer exists.

The carve-out name is the leading backticked token on each `-` bullet below ‚Äî keep
that format; the gate parses it.

## Host-policy behavioral fixtures

These programs validate engine **error/limit behavior**, not stdout, so they have no
`.expected_output.txt` and cannot be run by `dart run` (an infinite loop / unbounded
allocation / oversized program / sandboxed I/O). Each is asserted behaviorally by the
engine test harnesses instead ‚Äî `dart/engine/test/conformance_test.dart` and
`cpp/test/test_selfhost_conformance.cpp` (the `HostKnob` table).

- `196_timeout` ‚Äî `while(true){}`; engine must raise "Execution timeout exceeded" under `timeoutMs`.
- `197_memory_limit` ‚Äî unbounded `list_filled`; must raise "Memory limit exceeded" under `maxMemoryBytes`.
- `201_input_validation` ‚Äî oversized program; must raise "Too many modules" / "Expression too deep" / "Program too large".
- `202_sandbox_mode` ‚Äî `file_read` under `sandbox: true`; must raise "Sandbox violation".

## Tool-generated fixtures

- `256_editions_resolver` ‚Äî generated by `dart/encoder/tool/gen_editions_conformance.dart`
  (runs the real Editions feature resolver), not by the Dart-source encoder pipeline.

## Feature-gap carve-outs

- `188_std_time_now` ‚Äî exercises the `std_time` base module (`format_timestamp` /
  `parse_timestamp` over a fixed epoch). The syntactic encoder has no routing for
  `DateTime` / `dart:convert`-style time APIs to `std_time`, and the engine's
  `now()`-derived components are host-controlled (non-deterministic), so it cannot be
  generated from portable Dart source. Hand-authored; revisit if `std_time` encoder
  routing + a mockable clock are added.
- `389_typed_to_string` ‚Äî exercises `std.int_to_string` / `std.double_to_string`.
  These are type-specific variants of `to_string`; the syntactic encoders have no
  static type resolution, so `.toString()` always routes to the generic
  `to_string` and can never disambiguate by receiver type. Generated by
  `dart/encoder/tool/gen_std_gap_fixtures.dart` (issue #64 std-coverage gap).
- `390_goto_label` ‚Äî exercises the `label` / `goto` base functions (a backward goto
  loop printing `0,1,2`). No source language surfaced by an encoder has a
  `goto`/label statement, so it cannot be generated from portable source. Hand-authored
  (issue #184); it is the regression guard for the C++ self-host label loop ‚Äî the
  reified thrown `_FlowSignal` fix in `engine_control_flow.dart._gotoSignalLabel`.
- `399_bytes_literal` ‚Äî exercises a `Literal.bytes_value` node (the `literal.bytes_value`
  node-shape carve-out, #64 Phase 2b). No Dart source construct maps to a bytes literal
  (`Uint8List.fromList([...])` encodes as a constructor call), so it cannot be generated
  from portable source. Hand-authored; it is the regression guard for issue #244 (the
  TS and C++ compilers, plus the self-hosted TS engine, all discarded a bytes literal's
  real content before that fix).
