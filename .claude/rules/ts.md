# TypeScript-Specific Instructions

## Package Structure

Ball's TypeScript implementation has 5 packages (no workspace manager — each has its own
`node_modules`):

- `@ball-lang/shared` (`ts/shared/`) — Protobuf-es generated types from `ball.proto`. Uses
  `@bufbuild/protobuf` v2. Dependency for all others.
- `@ball-lang/compiler` (`ts/compiler/`) — Ball → TypeScript compiler using `ts-morph`.
  Generates declarations via structure API, expressions/statements as raw strings.
- `@ball-lang/encoder` (`ts/encoder/`) — TypeScript → Ball encoder (stub, under development).
- `@ball-lang/engine` (`ts/engine/`) — Self-hosted engine: `compiled_engine.ts` generated by
  compiling `dart/self_host/engine.ball.json`. Wrapper in `index.ts` provides proto3 JSON
  normalization and std function registrations. `run()` is async (returns `Promise<string[]>`).
- `@ball-lang/cli` (`ts/cli/`) — CLI entry point.

## Key Patterns

### Protobuf-es API (different from Dart protobuf)

```typescript
// Oneof discrimination
expr.expr.case === "call"         // Dart: whichExpr() == Expression_Expr.call
expr.expr.value                   // Dart: expr.call (after checking case)

// Presence checking
body !== undefined                // Dart: hasBody()
metadata?.["key"]                 // Dart: metadata.fields['key']

// Serialization
import { fromJson, toJson } from "@bufbuild/protobuf";
const program = fromJson(ProgramSchema, jsonObj);
const json = toJson(ProgramSchema, program);
```

### Compiler

- `compile(Program) → string` — returns TypeScript source
- Uses `ts-morph` for structural AST building (class/function declarations)
- Expressions/statements emitted as raw TS string into buffers
- Post-processing via regex for OOP field binding, super chain, cascade evaluation
- Preamble (`preamble.ts`) installs Dart-flavored polyfills on `Object.prototype`
- Base function dispatch in `_callBaseFunction()` switch

### Engine

- `compiled_engine.ts` — auto-generated, NEVER edit
- `index.ts` wraps it with proto3 JSON normalization (`protoWrap`) and method dispatch
- `run()` is async — returns `Promise<string[]>` (captured stdout lines)
- All base functions route through `std` — the `dart_std` module has been eliminated

### Encoder

- Under development — currently a stub
- Will use TypeScript Compiler API or `ts-morph` for AST parsing

## Generated Files — NEVER Edit

- `ts/shared/gen/` — Protobuf-es generated types
- `ts/engine/src/compiled_engine.ts` — Self-hosted engine compiled from Ball

To regenerate the compiled engine. Note `engine.ball.json` is a self-describing
`google.protobuf.Any` envelope (`{"@type":"…/ball.v1.Program", …}`), so the
`@type` key is stripped before compiling:
```bash
cd ts/compiler && node --experimental-strip-types -e "
const {readFileSync, writeFileSync} = require('fs');
const {compile} = require('./src/index.ts');
function unwrapBallFile(json){ if(json===null||typeof json!=='object'||Array.isArray(json))return json; const t=json['@type']; if(t===undefined)return json; const b={}; for(const[k,v]of Object.entries(json)){if(k!=='@type')b[k]=v;} return b; }
const program = unwrapBallFile(JSON.parse(readFileSync('../../dart/self_host/engine.ball.json', 'utf8')));
const ts = compile(program);
writeFileSync('../engine/src/compiled_engine.ts', '// @ts-nocheck — auto-generated\n' + ts);
"
```

## Testing

- Engine tests: `cd ts/engine && npm test` (vitest, conformance fixtures)
- Compiler tests: `cd ts/compiler && npm test` (node --test, fixture compilation + engine parse)
- Encoder tests: `cd ts/encoder && npm test`
- Prefer conformance tests over unit tests

## Build

```bash
cd ts/engine && npm ci && npm run build
cd ts/compiler && npm ci && npm run build
cd ts/encoder && npm ci && npm run build
```

## Dependencies

- `@bufbuild/protobuf` v2 — Protobuf-es runtime (shared)
- `ts-morph` — TypeScript AST manipulation (compiler)
- `vitest` — Test framework (engine)
- Node.js 22+ with `--experimental-strip-types` for running `.ts` files directly

## Publishing

- npm via OIDC trusted publishing (see `.github/workflows/publish-npm.yml`)
- Package names: `@ball-lang/shared`, `@ball-lang/compiler`, `@ball-lang/engine`, `@ball-lang/cli`
