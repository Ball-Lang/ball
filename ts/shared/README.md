# @ball-lang/shared

[Protobuf-es](https://github.com/bufbuild/protobuf-es) generated TypeScript types for the [Ball programming language](https://github.com/ball-lang/ball), generated by `buf` from `proto/ball/v1/ball.proto` (the single source of truth for the Ball IR). Provides typed messages with oneof discriminated unions, JSON/binary serialization, and presence checking. This is the shared dependency for `@ball-lang/compiler`, `@ball-lang/encoder`, and the other TypeScript packages.

## Install

```bash
npm install @ball-lang/shared @bufbuild/protobuf
```

`@bufbuild/protobuf` (v2) is the runtime that the generated schemas use for `fromJson`/`toJson`/`fromBinary`/`toBinary`.

## Usage

```ts
import { ProgramSchema, type Program } from '@ball-lang/shared';
import { fromJson, toJson } from '@bufbuild/protobuf';

// Parse a proto3-JSON Ball program object into a typed Program message.
const program: Program = fromJson(ProgramSchema, jsonObject);

// Oneof discrimination uses protobuf-es discriminated unions:
const expr = program.modules[0].functions[0].body!;
if (expr.expr.case === 'call') {
  console.log(expr.expr.value.function); // the called function name
}

// Serialize back to proto3 JSON.
const json = toJson(ProgramSchema, program);
```

### Self-describing ball files

A `.ball.json` / `.ball.bin` file is a `google.protobuf.Any` envelope wrapping exactly one top-level message (a `Program` or a `Module`), discriminated by its `@type` field. Helpers for reading/writing these envelopes are exported from the `./ball-file` subpath:

```ts
import { decodeProgramJson, encodeProgramJson } from '@ball-lang/shared/ball-file';

const program = decodeProgramJson(JSON.parse(fileContents));
```
