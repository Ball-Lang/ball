# ts/shared (`@ball-lang/shared`)

## Purpose

Provides protobuf-es generated TypeScript types for `ball.proto`, plus the `BallFile` envelope codec (`src/ball_file.ts`). All other TS packages depend on this one.

## Key Files

| File | Description |
|------|-------------|
| `gen/ball/v1/ball_pb.js` / `ball_pb.d.ts` | **Generated** — never edit. Typed message classes for `Program`, `Module`, `Expression`, etc. Entry export (`"."` and `"./proto"`). |
| `src/ball_file.ts` | Editable. Ball-file envelope codec: `unwrapBallFileJson`, `decodeBallFileJson`, `decodeProgramJson`, `encodeProgramJson`, `decodeProgramBinary`, etc. Mirrors `dart/shared/lib/ball_file.dart`. |

## For AI Agents

- `gen/` is **generated by `buf generate`** — never edit. Regenerate with `buf generate` at repo root after changing `proto/ball/v1/ball.proto`.
- Protobuf-es v2 API differs from Dart: oneofs use `expr.expr.case` / `expr.expr.value` (not `whichExpr()`); presence is `field !== undefined` (not `hasField()`); serialization uses `fromJson(ProgramSchema, obj)` / `toJson(ProgramSchema, program)` imported from `@bufbuild/protobuf`.
- `src/ball_file.ts` is the authoritative TS implementation of the `.ball.json` / `.ball.bin` envelope format (a `google.protobuf.Any` wrapper). Use `unwrapBallFileJson` when operating on raw proto3-JSON trees; use `decodeProgramJson` when you need a fully-typed `Program` message.
- Test runner: `node --experimental-strip-types --test test/*.test.ts`.
- See `.claude/rules/ts.md` for the full protobuf-es API mapping table and `CLAUDE.md` for invariants.

## Dependencies

- External: `@bufbuild/protobuf` v2 (runtime for generated types and serialization)
